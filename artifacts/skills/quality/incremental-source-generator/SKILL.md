# Incremental Source Generator (IIncrementalGenerator)

## Core Principles

- Implement `IIncrementalGenerator`, never the legacy `ISourceGenerator`.
- Start from `ForAttributeWithMetadataName` — it is dramatically cheaper than a hand-written `CreateSyntaxProvider` predicate.
- Every model that flows through the pipeline MUST be **value-equatable**: use `record`/`record struct` with primitives, `string`, and `EquatableArray<T>`. NEVER let `ISymbol`, `SyntaxNode`, or `Compilation` enter a pipeline value — they break caching and pin compilations in memory.
- Extract everything you need from symbols *inside the transform* into a plain data model; emit from that model only.
- Register output with `context.RegisterSourceOutput`; use `RegisterPostInitializationOutput` for the marker attribute so users do not reference your assembly.
- Target `netstandard2.0`; the generated code targets the consumer's TFM.

## Example

```csharp
[Generator]
public sealed class CommandRegistrationGenerator : IIncrementalGenerator
{
    private const string Marker = "MyApp.RegisterCommandAttribute";

    public void Initialize(IncrementalGeneratorInitializationContext context)
    {
        context.RegisterPostInitializationOutput(ctx => ctx.AddSource(
            "RegisterCommandAttribute.g.cs",
            "namespace MyApp { [System.AttributeUsage(System.AttributeTargets.Class)] " +
            "public sealed class RegisterCommandAttribute : System.Attribute { } }"));

        IncrementalValuesProvider<CommandModel> models = context.SyntaxProvider
            .ForAttributeWithMetadataName(
                Marker,
                predicate: static (node, _) => node is ClassDeclarationSyntax,
                transform: static (ctx, _) =>
                {
                    var symbol = (INamedTypeSymbol)ctx.TargetSymbol;
                    // Project into an equatable model — no symbols past this point.
                    return new CommandModel(
                        symbol.ToDisplayString(),
                        symbol.Name);
                });

        context.RegisterSourceOutput(models, static (spc, model) =>
        {
            var src = $$"""
                // <auto-generated/>
                namespace MyApp.Generated;
                public static partial class CommandRegistry
                {
                    public static void Register{{model.Name}}() { }
                }
                """;
            spc.AddSource($"{model.Name}.Registry.g.cs",
                SourceText.From(src, Encoding.UTF8));
        });
    }
}

// Equatable value model — record gives correct structural equality for caching.
internal readonly record struct CommandModel(string FullyQualifiedName, string Name);
```

## Cacheability Checklist

- Models are `record`/`record struct` with only equatable members.
- Collections use a custom `EquatableArray<T>` wrapper, not `ImmutableArray<T>` (whose default equality is referential).
- No `Compilation`, `ISymbol`, `SyntaxNode`, or `SemanticModel` survives the transform.
- Diagnostics are reported via the pipeline, not by throwing.
- Snapshot-test with `Microsoft.CodeAnalysis.Testing` and assert the driver reports `IncrementalStepRunReason.Cached` on a no-op edit.
